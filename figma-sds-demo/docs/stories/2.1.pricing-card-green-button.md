**I want** the pricing card action button to display in green,  
**So that** the call-to-action is more visually prominent and matches the updated brand design.

## Technical Implementation

### Recommended Approach: Add "positive" Button Variant

Create a new `.button-variant-positive` class in `button.css`:

```css
.button-variant-positive {
  --button-background-color: var(--sds-color-background-positive-default);
  --button-border-color: var(--sds-color-border-positive-default);
  --button-color: var(--sds-color-text-positive-on-positive);
  &[data-hovered]:not([data-disabled]) {
    --button-background-color: var(--sds-color-background-positive-hover);
    --button-border-color: var(--sds-color-border-positive-default);
  }
}
```

### Files to Modify

1. **`src/ui/primitives/Button/button.css`**
   - Add `.button-variant-positive` styles

2. **`src/ui/primitives/Button/Button.tsx`**
   - Add `"positive"` to the `variant` prop type
   - Add class mapping for the new variant

3. **`src/ui/compositions/Cards/Cards.tsx`**
   - Update default `actionVariant` to `"positive"` for PricingCard
   - Or update `pricingPlanToPricingCardProps` to use `"positive"` variant

4. **`src/figma/primitives/Button.figma.tsx`** (optional)
   - Update Code Connect mapping if Figma has a corresponding variant

### Design Tokens Used

| Token | Value | Usage |
